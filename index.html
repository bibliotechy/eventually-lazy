Unit Testing

  • Validates that your code does what it is supposed to do
  • Explicitly states your assumptions about what it does
  • Can help you plan your code before you write it
  • Encourages you to write small, reusable code chunks

Unit testing: an example

Is it actually a phone number?

  • 3 digits of area code, optionally surrounded by parenthesis
  • optional separator character (space or - or .)
  • 3 digits
  • optional separator character (space or - or .)
  • 4 digits

A solution 


function verify_is_phone_number($number) {
  $regex = '/^\(?(\d{3})\)?[\s-.]?(\d{3})[\s-.]?(\d{4})$/';
  preg_match($regex, $number, $matches);
  return $matches;
}

But how do you know it works?

A test


$valid_numbers = array(

foreach ($valid_numbers as $number) {
  $message = "{$number} is not a valid number";

Some other benefits

  • A document of what how your code works
    (Don't have to decipher the regex each time)
  • A baseline for when things change

Test Driven Development

Write a failing test, then make it pass.

Think about and plan each line of code before you write it

Unit Testing Drawbacks

  • Initially, it slows down development
  • Hard to add testing to existing applications
  • If you write the tests, your tests have the same blind spots as your code

Functional Testing

  • Testing that the user interface does what you expect
  • Testing user interaction

Great for sanity checks like:

  • Do the important forms work? Are they validating input?
  • Do the important links work?
  • Can users login?
  • Are anonymous users prevented from seeing private content?

Functional tests can be brittle, especially with older html (I'm looking at you, opacs!)

Options for functional testing

  • Selenium : Cross Browser testing, for realz! But, lots of set up. FF lets you record tests to run later on many browsers.
  • CasperJS: Webkit only, local, fast, easy to write. Won't catch cross browser issues, but will help you realize a change would prevent users from logging in!
  • Testling + Browserling - Cross browser Javascript testing as a service - Write javascript, test every browser. But, you gotta pay to run the tests. Useful if you are doing lots of complex JS, like client side MVC.
  • Many, many more options

Continuous Integration - CI

  • Regularly integrate little changes into working code to find and fix bugs faster
  • Really useful when more than one person is working on the same codebase
  • Leverage those tests and automated builds on a daily basis.

But why?

  • Helps establish a workflow for getting code from test to production
  • Makes the workflow visible
  • Logs for your changes. Know when things happened.

CI Options

  • Jenkins-CI: Free, flexible, very popular, but you run it yourself, and can be complex.
  • Travis-CI: Hosted, quick to set up, free for public-repositories. Requires your code to be on github
